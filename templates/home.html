Requirement Writing Guide & AI Capabilities

✓ DO's - Write Clear Requirements

  • Use action verbs: "Users should be able to register", "Add products to cart", "Remove items from cart" - AI automatically extracts these actions and generates contextual test steps
  • Be specific about constraints: "Password must contain 8+ characters, 1 uppercase, 1 number, 1 special character" - AI detects constraints and generates boundary tests with correct units
  • Include numbers with units: "Maximum 5 items per cart", "Minimum 10 characters", "Expire after 24 hours" - AI auto-detects units (items, characters, hours) for smart boundary testing
  • Mention error cases: "If payment fails, display error message" - AI generates functional tests for error scenarios
  • Use consistent format: Bullet points or numbered lists work best for requirement extraction
  • Include validation rules: "Email must be unique and valid format" - AI generates validation and edge case tests with realistic test data
  • Specify entities clearly: "Users", "products", "items", "orders", "messages" - AI extracts entities for contextual test data generation
  • Mention business rules: "Generate confirmation number after successful order" - AI creates data consistency tests

✗ DON'Ts - Avoid These Mistakes

  • Avoid vague statements: "System should work properly" - AI needs specific actions and entities
  • Don't use ambiguous language: "User should be able to do something" - Be specific about what action and what entity
  • Avoid implicit assumptions: Assume nothing about domain knowledge - AI works across all domains
  • Don't mix multiple requirements: Keep one requirement per statement for better test case generation
  • Avoid generic descriptions: "The system should be good" - AI needs measurable, specific requirements
  • Don't skip validation rules: Always mention format/length requirements - AI uses these for validation tests
  • Avoid unclear abbreviations: Use full terms so AI can extract entities correctly
  • Don't include error messages as requirements: "Please try again after 15 minutes" is a message, not a requirement

📋 Template Examples

✓ Functional Requirement (Action-Based):

"Users should be able to add products to the shopping cart. The maximum number of items in the cart should not exceed 10."

AI generates: Positive test (add product), Negative test (invalid product), Boundary test (10 items limit), Functional test (add operation)

✓ Validation Requirement:

"The email address must be unique and follow valid email format. The password must contain at least 8 characters, including one uppercase letter, one number, and one special character."

AI generates: Validation tests (email/password format), Boundary test (8 character limit), Edge case test (special characters), Security test (malicious input)

✓ Business Rule:

"The reset link should expire after 24 hours. If no results are found, the system should display a message 'No products found'."

AI generates: Boundary test (24 hours), Message test (exact message display), Data consistency test

🤖 AI Capabilities - What the System Does

🎯 Dynamic Action Extraction

AI identifies action verbs (add, remove, create, delete, update, validate, etc.) and generates contextual test steps specific to each action.

Example: "Add products" → Test steps include: Prepare product, Add operation, Verify acceptance, Confirm success

🏷️ Entity Detection

AI extracts entities (users, products, items, orders, messages, etc.) and uses them in test data and expected results.

Example: "Remove items from cart" → Test data: "Valid items", Expected: "System successfully removes the items"

📊 Smart Test Type Selection

AI automatically determines which test types are relevant based on requirement keywords:

  • Email/Password → Validation + Security tests
  • Numbers/Limits → Boundary tests
  • Unique/Duplicate → Edge case tests
  • Messages → Message display tests
  • Save/Store → Data consistency tests

🔢 Automatic Unit Detection

AI detects units from requirements and generates boundary tests with correct units:

  • "8 characters" → Boundary test: "At limit: 8 characters, Exceeds: 9 characters"
  • "24 hours" → Boundary test: "At limit: 24 hours, Exceeds: 25 hours"
  • "10 items" → Boundary test: "At limit: 10 items, Exceeds: 11 items"

💾 Realistic Test Data Generation

AI generates domain-specific test data based on requirement context:

  • Email fields → user@example.com, test@domain.co.uk
  • Password fields → SecurePass123!, MyP@ssw0rd
  • Quantities → 1, 5, 10
  • URLs → https://example.com, https://test.co.uk
